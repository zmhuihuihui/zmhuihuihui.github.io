详解Java中的Synchronized原理
Synchronized
Java
发布日期:   2020-01-03
文章字数:   889
阅读时长:   3 分

Synchronized的基本使用

Synchronized是用来实现对临界资源的互斥访问的，有三种方式可以来实现同步：

  • Synchronized作用于普通方法上，即对当前this对象加锁
  • Synchronized作用于静态方法上，即对整个Class对象加锁
  • Synchronized作用于代码块上，即对括号里的对象加锁
class SynchronizedDemo{

    //锁住当前this对象
    public synchronized void method1(){
        System.out.println("方法一");
    }

    //锁住整个Class对象
    public synchronized static void method2(){
        System.out.println("方法二");
    }

    //锁住当前this对象
    public void method3(){
        synchronized (this) {
            System.out.println("方法三");
        }
    }

    //锁住整个Class对象
    public void method4(){
        synchronized (SynchronizedDemo.class) {
            System.out.println("方法四");
        }
    }
}

上述的锁分为两种，一种为对象锁一种为类锁：

  • 对于对象锁，只能锁住当前对象的synchroiezd块，保证多个线程访问同一个对象的synchroiezd块时同步。

  • 对于类锁，只能锁住整个类的的synchroiezd块（注意并不是锁住整个class类，其他方法仍让保持异步），保证多个线程访问不同对象的synchroiezd块时同步。

不管是对象锁还是类锁，都不能锁住该类的非synchroiezd块。

锁机制原理

当一个线程访问临界资源时，需要获取锁才能进行访问；当访问结束时必须释放该锁，那么Synchronized是如何实现锁机制的呢？

首先对于每一个对象在创建时，JVM中都会同时创建monitor（监视器锁）。当monitor被占用时就会将此对象锁定。当将Synchronized作用于代码块时，运行的线程会在代码块之前先执行monitorenter指令尝试获取monitor，如果获取成功继续执行；如果获取失败则阻塞，等monitor被释放后再尝试获取。当运行的线程执行完代码块之后，则会执行monitorexit指令释放monitor。

锁的优化

从JDK6开始，对Synchronized进行了优化，极大的提高了性能。其定义了四种锁的状态：无锁状态、偏向锁状态、轻量级锁状态、重量级锁状态。这些锁只能向上升级并不能向下降级。

  • 偏向锁
    目的：在没有多线程竞争的情况下，只有一个线程访问同步代码块时，可以提高执行效率。
    如果一个线程获得了锁，那么该线程就会从无锁状态切换到偏向锁状态，这一步是CAS操作完成的。在偏向锁状态下，该线程每一次访问同步代码块都不需要进行同步操作，减少了同步获取锁和释放锁的性能消耗，提高了效率。

  • 轻量级锁
    目的：在没有发生竞争的情况下，减少传统重量级锁在使用互斥量时产生的性能消耗。
    当多个线程进入同步代码块，但未发生竞争时，就会通过CAS来获取轻量级锁，将少了锁状态的切换。当有多个线程产生竞争时，首先或通过CAS自旋来获取锁，自旋在极短的时间内发生，并有固定的自旋次数，如果自旋获取锁失败，则将会升级为重量级锁。因此，轻量级锁适用于线程交替执行同步代码块的场景。

  • 重量级锁
    重量级锁通过监视器锁（Monitor）来实现同步的，此过程需要操作系统从用户态切换到核心态，转换时间较长成本较高，因此效率较低。
